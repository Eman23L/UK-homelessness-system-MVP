# Data Governance Rules

## Data Sources
- Only use publicly accessible websites
- Do not scrape login-protected or private content
- Respect robots.txt where applicable

## Personal Data Protection
- Do NOT store personal staff names
- Do NOT store personal emails or phone numbers
- Only collect organisation-level contact information

## Data Tracking
- Every entry must include source_url
- Every entry must include date_collected

## Verification
- All entries default to "unverified"
- Only verified entries should be shown in the app
- Manual review required before publishing

## Accuracy & Responsibility
- Data must not mislead users
- Clearly indicate if information is outdated or unverified
- Prioritise safety-critical accuracy
